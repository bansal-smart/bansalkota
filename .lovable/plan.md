# Cashfree Payments Integration Plan

Goal: replace the current "place order / contact you for payment" placeholders with a real **Cashfree Payment Gateway** flow (UPI / Cards / Netbanking / Wallets) for:
1. E-Store cart checkout (`CheckoutPage.tsx`)
2. Course enrollment (`EnrollmentModal.tsx`)
3. Test Series enrollment (`TestSeriesDetailPage.tsx`)

We will use Cashfree's **Hosted Checkout (Payment Sessions API v2025-01-01)** — the user is redirected to Cashfree's secure page, pays, then is redirected back. This is the safest pattern (no PCI burden, no Drop-in JS surprises).

## 2. Database changes (one migration)

Extend existing `orders` and `payments` tables (no new tables needed — both already exist):

- `orders`: add `cf_order_id text`, `cf_payment_session_id text`, ensure `status` supports `pending | paid | failed | cancelled`.
- `payments`: ensure columns `provider text` (`cashfree`), `provider_order_id`, `provider_payment_id`, `amount`, `currency`, `status`, `raw_payload jsonb`, `order_id` FK.
- Add unique index on `payments(provider, provider_payment_id)` for webhook idempotency.
- GRANTs preserved; RLS already in place.

For **course** and **test-series** purchases (currently not going through `orders`), we will route them through the same `orders` table with `item_type = 'course' | 'test_series'` in `order_items` so one payment flow serves all three.

## 3. Edge functions (3 new)

### a) `cashfree-create-order`
Input: `{ orderType: 'cart'|'course'|'test_series', items, shipping?, courseId?, testSeriesId? }`
- Auth-gated (verify JWT in code).
- Computes total server-side from DB (never trust client price).
- Inserts row in `orders` + `order_items`.
- Calls Cashfree `POST /orders` with our `order_id`, amount, customer details (from `profiles`), and `return_url = {site}/payments/return?order_id={id}`.
- Stores `cf_order_id` + `payment_session_id` back on the order.
- Returns `{ payment_session_id, cf_order_id, order_id }`.

### b) `cashfree-webhook` (public, signature-verified, `verify_jwt = false`)
- Verifies `x-webhook-signature` HMAC-SHA256 using `CASHFREE_WEBHOOK_SECRET` + raw body + timestamp header.
- Handles `PAYMENT_SUCCESS_WEBHOOK`, `PAYMENT_FAILED_WEBHOOK`, `PAYMENT_USER_DROPPED_WEBHOOK`.
- Upserts `payments` row (idempotent on `provider_payment_id`).
- On success: marks `orders.status = 'paid'`, creates `enrollments` row if course/test_series, sends `payment-receipt` email (existing template) + notification.
- Returns 200 quickly.

### c) `cashfree-verify-order` (called by return page as a fast path)
- Server-side `GET /orders/{order_id}/payments` to confirm status before showing success UI (in case webhook is delayed).

## Technical notes
- All amounts in INR, integer paise avoided — Cashfree accepts `order_amount` as decimal rupees.
- Server is source of truth for price; client-sent amounts are ignored.
- Webhook signature: `base64(HMAC_SHA256(timestamp + rawBody, secret))` compared in constant time.
- Edge functions use `npm:@supabase/supabase-js@2`, no Cashfree SDK (raw `fetch` against REST API — keeps cold starts fast and avoids Deno/npm SDK issues).
